### 一、标题

#### 典型结构：可从Google学术、知网寻找好的标题，进行替换对应词语转换成自己的标题

```
1. 基于 [方法名称] 的 [研究对象] [应用功能] 方法
   例：基于模型预测控制的虚拟同步发电机控制策略方法
2. [提出的模型]：[研究对象]的[该模型的功能]
   例：SolarNet：面向分布式光伏的超短期概率预测方法
3. 面向[应用场景/特定需求]的[研究对象][研究类型]
   例：面向边缘计算环境的轻量化神经网络入侵检测方法
```

#### 禁止事项：

- 避免出现多个“的”连接词，会显得标题比较臃肿

---

### 二、摘要

**目的/问题：**一句话交代明白本文解决的核心问题/目前研究存在的缺陷，思考一下自己写的这个问题能够被后文工作所解决，若不能→改

```
为解决……问题，本文提出……。
现有……方法存在……不足，因此本文旨在……。
```

**方法/过程：**最基础的摘要采用【首先、其次、最后】来阐明本文方法的每一步工作，这是给审稿人最直观展示你做的工作的**流程**

```
首先，引入了……理论/设计了……框架；其次，建立了……模型/设计了……算法；最后，通过……进行验证。[不局限于这几种论述形式]
```

**结果/发现：**不允许泛泛而谈结果更优、更精确，要论述自己方法**新的功能**或**提升的性能**

```
结果表明，所提方法能够……，相较于……方法，……指标提升了……%。
实验显示，在……工况下，……误差降低了……，响应时间缩短了……。
```

**结论/意义：**可选项，说明本文研究的适用范围

```
本研究为……提供了新的解决思路，具有……理论意义/工程价值。
本文工作表明，……理论具有可行性，可推广至……场景。
```

#### 禁令/注意事项：

- 禁止在正文中出现类似的首先、其次、最后的摘要描述结构
- 摘要中除自己所提方法以外，不得出现新造的词，会给读者带来阅读障碍

---

### 三、引言

**第一段：大背景与研究意义**：需要在背景收尾阶段引出本文解决的核心问题

```
随着……的快速发展，……在……领域得到了广泛应用。然而，……问题日益突出，严重影响了……。
近年来，……已成为研究热点，其中……是保障系统安全稳定运行的关键。
```

**第二段：文献综述（分类评述）**：将文献综述划分为2-3段，从不同的研究角度进行分类描述，以新能源预测为例，可从时间尺度、协变量数据、预测结果形式、预测模型等方面论述。以电碳协同调度为例，可从多时间尺度协同机制、不确定性优化方法、具体的降碳方法（碳捕集、市场机制、多能源转换）等方面论述。思考自己论文的**关注点/创新点**在何处，选择对应的分类进行国内外研究的阐述，最后写出这个分类标准下的**研究不足**。这个**不足需要在本文中被解决，不能只管提出问题，但不解决！**

```
研究角度1：
关于……问题，国内外学者已开展大量研究。在……方面，文献[1-3]提出了……，但其存在……不足。在……方面，文献[4-6]通过……取得了……效果，但未考虑……。[总结]值得注意的是，……研究相对较少，尤其是……仍缺乏有效解决方案。
研究角度2：
关于……问题，国内外学者已开展大量研究。在……方面，文献[1-3]提出了……，但其存在……不足。在……方面，文献[4-6]通过……取得了……效果，但未考虑……。[总结]值得注意的是，……研究相对较少，尤其是……仍缺乏有效解决方案。
```

**第三段：指出空白与问题**：如果在文献综述中未进行问题总结的，可以单独列一段描述问题。

```
综上所述，现有研究存在以下不足：
（1）多数方法忽略了……；（2）……依赖于离线整定参数，适应性差；（3）……缺乏对……约束的显式处理。
因此，迫切需要一种……的方法。
```

**第四段：本文工作与贡献**：贡献不包含方法的性能，说明本文方法的核心创新，与众不同之处

```
针对上述问题，本文提出……。主要贡献如下：
（1）建立了……模型，实现了……；
（2）设计了……算法，解决了……；
```

**第五段：论文结构**：英文论文需要介绍后续章节

```
本文组织结构如下：第1节介绍……；第2节给出……模型；第3节阐述所提……策略；第4节通过仿真验证……；第5节总结全文并展望未来工作。
```

#### 禁令/注意事项：

- 综述部分大量出现“张三等[1]做了……；李四等[2]做了……”，进行论文排列性的论述，没有分门别类进行总结分析。
- 本文工作与贡献不能与摘要相似，需要凝练本文与现有文章的不同，而并不论述本文的研究流程

---

### 五、总体框架/基础知识/基础问题

if 论文引入了一种新的理论体系[例如引入了**最优传输理论**去优化生成式路径]，可以进行**Fundamentals**/**Preliminaries**的介绍，但需要注意该理论在应用场景是新颖的，禁止出现做调度优化，开头介绍最优潮流。

if 论文并没有引入新的理论体系，可以在此描述一个新的研究框架[例如提出了一个全新的多时间尺度电碳协同调度框架]，可以对框架中的流程进行具体介绍，让读者理解此框架与现有框架的差异。

if 论文是开展了一个新的数学问题研究，可以在此用公式说明本文解决的问题[例如提出了一个综合考虑多源NWP数据的长期新能源场景生成问题]，可定义输入、输出、代数方程组、用神经网络拟合的函数，综合来说明解决的数学问题。

#### 禁令/注意事项：

- 本段描述与全文论述问题不对应，这部分属于全文的索引or基础，是在摘要之后更为详细些了解全文的窗口。

- 建议添加最少一张图，图比文字更容易让人直观理解，但禁止在此画一张巨大的图去包含所有的技术细节。

### 七、所提模型/方法

这部分是全文技术细节的描述，分解描述本文模型

1-可参考摘要中列些的三部分，每一个部分进行模型、算法的详细描述

2-可依据模型拆解的几个部分，进行总-分描述

#### 禁令/注意事项：

- 每一个二级标题为通用化的描述，没有加任何限定词，例：2.2 约束调节， 2.1 目标函数，无法体现本文的差异化工作，需要具体。
- 禁止大篇幅描述基础性概念，例如交流潮流非本文核心改进工作，进行科普性描述
- 公式符号需要保持一致，不允许出现一个对象前后有两个符号

----

### 八、算例/仿真/实验

**子标题示例：**撰写算例时做好算例涉及，用什么样的数据实现本文方法的验证，每个Case画什么图、什么表，说明什么样的结论，**此结论能否与本文的贡献对应？**

```
4.1 算力概况/仿真平台与参数设置
4.2 Case1 : 给出算例的名称，是分析了什么内容
4.3 Case2 : 给出算例的名称，是分析了什么内容
4.4 Case3 : 给出算例的名称，是分析了什么内容
```

#### 禁令/注意事项：

- 只放图，不在正文中分析图
- 表格中出现的数据不能在正文中反复说明，任何图片与表格需要分析其产生此结果的内在原因，此原因是本文工作所产生的效果
- 缺少对比实验，没有使用经典方法作为基准

### 九、总结与展望

**研究总结：**2-3个结论

```
本文针对……问题，提出了……方法。通过理论分析和仿真验证，得出以下结论：
（1）提出了……，说明在算例中产生的效果（效果部分是在引言-贡献部分没有描述的，此部分是在得到算例结果后的总结）；
（2）……；
（3）……。
```

**贡献回顾：**

```
本文的主要贡献在于：提出了……；揭示了……；验证了……。
```

**局限性（如需）：**

```
需要指出的是，本文方法在……方面仍有不足，例如……。
```

**未来工作：**

```
后续研究可从以下方向展开：
（1）将所提方法扩展至……场景；
（2）引入……技术进一步提升……性能；
（3）开展半实物/现场实验验证。
```

#### 注意事项：

- 结论中不要出现新的图表或文献
- 不要照抄摘要中的句子
- 每个结论都要有正文结果支持

### 十、参考文献

#### 格式示例：注意中英文期刊论文对参考文献的公式规范

```
[1] 时珊珊, 鲁宗相, 闵勇, 等. 微电网孤网运行时的频率特性分析[J]. 电力系统自动化, 2011, 35(9): 36-41.
[2] Zheng Y W, Chen M Y, Li C, et al. A microgrid control strategy with adaptive droop coefficient[J]. Automation of Electric Power Systems, 2013, 37(7): 6-11.
[3] 石荣亮, 张兴, 徐海珍, 等. 基于虚拟同步发电机的多能互补孤立型微网运行控制策略[J]. 电力系统自动化, 2016, 40(18): 32-40.
```

#### 注意事项：

- 英文文献中尽量少用中文参考文献
- 注意同期刊的参考文献往往有可能是审稿人团队，引用过程需要措辞严谨，避免产生误会
- 每一篇参考文献都要在正文中被标记
